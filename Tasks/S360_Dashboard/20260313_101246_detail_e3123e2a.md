## KPI Meaning

This KPI is `[SFI-NS2.2.1] Secure PaaS Resources`. It applies to public cloud PaaS resources such as Storage, Key Vault, SQL, and Cosmos DB. For the Azure SignalR Service scope, the currently affected items are all **Key Vault** items.

The KPI is considered compliant if **any one** of the following is true:

1. Public network access on the resource is set to `Disabled`, with access moved to Private Link / Private Endpoint.
2. The resource is associated with a Network Security Perimeter (NSP).
   For this service scope, the relevant action item subtype is `Multiwave 5-6: Secure PaaS Resources Learning mode`, which expects **Transition mode** for non-HVT/TCB services.
3. The resource has an approved S360 extension or exception.

## What Needs To Be Done

- Review each affected Key Vault and remove public exposure.
- Preferred end state: either disable public network access entirely, or attach the vault to an NSP so it is no longer considered publicly reachable.
- Because this service is tagged `Non-HVTTCB`, the expected NSP posture here is **Transition mode** rather than the HVT/TCB enforced-mode requirement.
- If a vault cannot be remediated before the due date, an approved extension/exception is the only compliant fallback.
